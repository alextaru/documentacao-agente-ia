# Templates para Features

Cada criação ou atualização de arquivo deve seguir o seu template correspondente.

## Templates corespondem

- `FEATURE.MD`: template para criação de nova feature, deve ser seguido para criar novas features na pasta `features/`.
- `FEATURE-REGISTRY.yaml`: modelo do arquivo de registro de features, deve ser seguido para atualizar ou criar o registro de features em `feature-registry.yaml`.
